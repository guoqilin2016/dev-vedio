# 飞书 CLI 视频策划

## 核心结论

飞书不是又加了一个 AI 按钮，而是在把自己变成 AI Agent 可以真正调用的协作入口。

## 目标受众

- 泛 AI 用户
- 想看“AI 真正干活”而不是只看聊天的人
- 对国内协作工具和 Agent 落地感兴趣的创作者与从业者

## 口径边界

- 可以明确讲：官方 OpenAPI MCP / CLI 已经把文档、消息、多维表格、任务、日历等大量协作能力接进本地 Agent 工作流
- 不能绝对化讲：飞书已经把所有远程能力都零门槛完全开放
- 要保留校正：官方远程 MCP 文档公开口径更保守，Docs 是最清晰的公开场景，本地 CLI / OpenAPI MCP 的能力覆盖更广

## 7 段结构

1. Hook：AI 不只是聊天，已经开始接管飞书里的真实工作
2. Positioning：飞书不是多了 AI 功能，而是在变成 Agent 工作入口
3. Proof 1：Markdown / 资料 → 飞书文档
4. Proof 2：一句话安排日程 → 查忙闲 → 建会议
5. Proof 3：文档 / 数据 → 多维表格 / 看板
6. Nuance：远程公开口径保守，但本地 OpenAPI MCP / CLI 覆盖很广
7. CTA：未来竞争不是谁更会聊天，而是谁能让 Agent 真做事

## 画面方向

- Scene 1: CLI + 聊天动作叠加，前 2 秒直接给“AI 开始接管飞书工作”
- Scene 2: 飞书工作入口的桥接图
- Scene 3: 终端指令转成文档结果页
- Scene 4: 忙闲网格、会议安排、日程卡片
- Scene 5: 多维表格、状态列、看板卡片
- Scene 6: 双栏对比，左边公开 Docs 口径，右边本地 CLI / MCP 覆盖面
- Scene 7: 强结论 + 标签 + 行动号召

## 标题方向

- 飞书开始给 Agent 开门
- AI 开始接管飞书工作
- 飞书 CLI 真正的意义

## 画面风格

- 深色协作科技风
- 主色偏飞书蓝与青色
- 高对比信息卡、终端窗口、文档页、日历格和看板列轮换
